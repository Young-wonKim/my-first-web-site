생활코딩

생활코딩 수업을 듣고 JavaScript를 실습해본다.

수업소개, JS소개 (1-2장)

  • JavaScript : 사용자와 상호작용 할 수 있도록 기능을 추가하는 언어이다.(동적) ↔ html(정적)
  • 검사(inspect) - element = tag를 뜻함
  • onclick의 속성(attribute)으로는 JS가 와야함
  • onclick 속성의 값으로 JS를 넣어주면, 사용자가 onclick 속성이 위치하고 있는 버튼을 클릭했을 때, 그 값의 JS코드를 실행시킴

HTML과 JS의 만남 : script tag (3장)

  • JavaScript : HTML 위에서 동작하는 언어(JS의 출발점)
  • Script tag : 웹브라우저에게 HTML의 코드로 '지금부터 JS가 시작됩니다라고 알려주는 것
  • document.write : 웹페이지에 글씨를 출력할 때

HTML과 JS의 만남 : 이벤트 (event) (4장)

  • 이벤트 (event) : 웹브라우저 위에서 일어나는 일들. on~ 속성. JS가 사용자와 상호작용하는 데 핵심적인 역할
  • input tag : 버튼을 만들때 쓰는 tag. type을 button으로 하면 버튼 모양이 됨
  • 경고창 : alert 기능

HTML과 JS의 만남 : 콘솔 (console) (5장)

  • 검사 - 콘솔
  • 매우 실용적 : 파일이 아니라도 어떤 코드를 실행해야하는 가벼운 상황들 → 파일을 만들지 않고도 JS코드를 즉석에서 실행가능!
  • 새로운 웹사이트 개발 뿐만아니라, 이미 만들어져 있는 웹사이트를 나의 필요에 따라 맥락적으로 간단하면서 내게 딱맞는 코드를 이용해 문제를 해결 할 수 있게함
  • 콘솔에서 실행시키는 JS는 해당 웹페이지 안에 삽입된 JS인 것처럼 동작 (해당 웹페이지를 대상으로 실행)
  • .length : 문자열의 문자갯수를 알려주는 명령
  • 수업 중 마음에 박힌 문장 : '너무 큰 목표를 설정하는 것보다는 나의 현실의 문제를 가볍게, 가볍게 해결하는 방법을 잘 찾아 보셨으면 좋겠다는 생각이구요'

JS의 문법 : 데이터타입 data type(자료형) : 문자열string과 숫자number (6장)

  • 콘솔창 : 숫자는 그대로 쓰면 됨, 문자는 따옴표로 표시 ex)'hello' or "hello"
  • 검색 : mozilla site에서 검색
  • 1+1=2, "1"+"1"="11" : 문자와 숫자는 다른 데이터타입! 문자냐, 숫자냐에 따라 그 결과는 많이 달라질 수 있기 때문에 항상 데이터 타입을 정확하게 표현하는 것에 대해서 관심을 갖고 잘 할 필요가 있다.

JS의 문법 : 변수와 대입 연산자 (7장)

  • 변수variable (var), 상수constant
  • 실행을 유보하고 싶을때는 shift + enter

웹브라우저 제어 (8장)

  • 속성을 주고자 하는 태그가 누구인가하는것을 웹브라우저에게 JS문법으로 어떻게 하는지를 알아야 함
