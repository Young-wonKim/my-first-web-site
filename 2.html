웹페이지를 아름답게 디자인하고싶어! 새로운 욕망 (2장,3장)

웹페이지를 아름답게 디자인하는 갈림길
  • html의 문법에 태그를 추가 - 쉬운길 : 1)비효율적, 2)웹의 정보의 질을 떨어뜨림
  • CSS - 어렵지만 근본적인 해결책 : 1)정보를 가치를 높여줌 - html에서 디자인의 기능을 뺏어옴, 2)효율적(폭발적 효과!) 왜? 중복을 제거!
    웹페이지의 사이즈 자체를 줄여줘서 수정의 효율(유지보수) 뿐만아니라, 웹페이지 규모가 커질수록, 그 처리(속도)에도 큰 효율이 나타난다.(+가독성)
style tag : html의 문법이면서 동시에 다음과 같은 뜻을 지닌다 "웹브라우저야, 이 style tag 안쪽의 내용은 CSS니까, 넌 이것을 CSS라는 언어의 문법에 맞게 해석해서 처리해야해"
디자인과 관련된 코드는 style tag안에 다 넣어 버림으로써, 이 웹의 정보(내용)에는 디자인과 관련된 코드는 단 한줄도 포함하지 않게 된다. 그럼으로써 웹 페이지가 해석될 때, 정보만을 가진 코드들을 분석해서 여러가지 작업을 각자의 의도에 맞게 해석할 수 있다. (html이 정보에 전념가능, 정보의 가치를 높여줌)

CSS의 기본 문법 (4장)

웹페이지에 CSS를 포함시키기 위해서는 html과 CSS가 완전히 다른 언어이기 때문에 웹브라우저로 하여금 어디서부터 어디까지가 CSS인지 알 수 있게 해줘야 한다.
  1. style tag를 이용
  2. style 속성을 이용 : 적용부분에 직접 적용, 선택자(selector)를 사용할 필요 없음
tip) style tag사용시, 각 효과적용후 디스크립션이 시작되고 끝나는 것을 구분하기 위해서 구분자로 세미콜론(;)사용 (여러 효과가 한 줄에 있어도 잘 동작됨)
